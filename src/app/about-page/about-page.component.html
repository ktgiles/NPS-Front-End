This National Parks Tracker utilizes Java Spring Boot to access data directly from the National Park Service API and to create and maintain a SQL database for storage and manipulation of user accounts. The front end of the application was created with Angular, relying heavily on the Angular Material UI component library.

Possible future improvements include the following:
  • Implement Search by Park Name functionality
  • Implement Search by State functionality
  • Add features available only to logged in users:
    • Allow users to save parks for Future Trips/view all Saved parks
    • Allow users to mark parks as Visited/view all Visited parks
    • Allow users to save reviews and photos linked to their Visited parks
    • Allow users to see other users' reviews and photos
  • Improve site security:
    • Set up password requirements/validation
    • Set up password reset feature
    • Enable JWT authentication
    • Hide password on login page
  • Improve UI/UX with optional dark mode
  • Test UI for accessibility compliance
  • Add features to improve application usefulness to target market:
    • Note parks that require advance entry reservation and link to reservation site
    • Add user profile customization (profile picture, "about me", etc)
    • Allow users to connect with external social media sites (facebook, instagram, twitter, etc) to share Saved Parks and Visited Parks

  • Bug Fixes:
    • Search by Park Name/Button alignment changes breaks on large screen size
